The STATE in Ethereum refers to the current information or status of all accounts and smart contracts, and is a representation of the CUMULATIVE outcomes of all transactions and smart contract executions up to the current block. It's dynamic and changes with every new block.

The HISTORY on Ethereum, on the other hand, refers to the record of all transactions and blocks that have been processed and confirmed on Ethereum since genesis. This includes details like transaction data, block headers, receipts, etc.. The history is essentially a log of how the chain has evolved over time and how the current state has been reached. It's a comprehensive record but is not directly required for processing new transactions.

Summary

The primary aim is to prune historical block headers, bodies, and receipts that are older than one year from the clients. This move is motivated by the growing disk space requirement for storing Ethereum's full transaction history (in PoS Ethereum, SSD storage is one of the more expensive pieces of hardware).

State Expiry

×

Status

Ongoing research

Summary

State Expiry would remove or archive older state data from the active state of the Ethereum blockchain. Currently Ethereum state keeps growing as it accumulates all account balances, contract code, and contract storage from the genesis block. State Expiry aims to move older state data, which is not frequently accessed or modified, into a less readily accessible form of storage, effectively 'expiring' it from the active state. To do this we would categorize state data based on epoch age.

This contrasts with History Expiry, which focuses on pruning historical block data rather than the state data itself.
