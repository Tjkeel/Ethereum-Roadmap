The STATE in Ethereum refers to the current information or status of all accounts and smart contracts on the blockchain. This includes account balances, contract code, and contract storage. The state is a representation of the cumulative outcomes of all transactions and smart contract executions up to the current block. It's dynamic and changes with every block added to the blockchain. State data is crucial for the network to process new transactions and interactions correctly, as it reflects the current 'truth' of the network.

The HISTORY on Ethereum, on the other hand, refers to the record of all transactions and blocks that have been processed and confirmed on the blockchain since its inception. This includes details like transaction data, block headers, and receipts. The history is essentially a log of how the blockchain has evolved over time and how the current state has been reached. It's a comprehensive record but is not directly required for processing new transactions.

State Expiry

×

Status

Ongoing research

Summary

State Expiry in Ethereum refers to the proposal to systematically remove or archive older state data from the active state of the Ethereum blockchain. Currently Ethereum state keeps growing as it accumulates all account balances, contract code, and contract storage from the genesis block. State Expiry aims to move older state data, which is not frequently accessed or modified, into a less readily accessible form of storage, effectively 'expiring' it from the active state.

The implementation of State Expiry involves categorizing state data into different epochs based on age, with the older epochs being periodically archived. This archived data would still be accessible, but not stored in the active state, significantly reducing the amount of data a node needs to manage and process on a regular basis.

This contrasts with History Expiry, which focuses on pruning historical block data rather than the state data itself.
